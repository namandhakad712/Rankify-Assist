## ğŸ“‹ Guidelines

### Code Style

- **JavaScript**: Use ES6+ syntax, consistent indentation (2 spaces)
- **C/C++**: Follow TuyaOpen SDK conventions
- **Comments**: Explain WHY, not WHAT
- **Naming**: Use descriptive names (camelCase for JS, snake_case for C)

### Commit Messages

Format: `<type>: <description>`

Types:
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation changes
- `refactor`: Code refactoring
- `test`: Adding/updating tests
- `chore`: Build/config changes

Example: `feat: add support for custom LLM providers`

### Pull Request Process

1. Update README.md if needed
2. Add tests if applicable
3. Ensure all existing tests pass
4. Update docs/CHANGELOG.md
5. Request review from maintainers
